## Sec. 1 Building the Basic Version

The basic version includes the core TPBench library and the `stream` kernel (the default kernel for memory bandwidth evaluation). By default, only `tpbk_stream` is compiled to minimize build time and dependencies.

### Prerequisites

- CMake >= 3.16
- C11 compatible compiler (GCC, Clang, or Intel C Compiler)
- Linux operating system

### Basic Build Steps

```bash
# Clone the repository
git clone https://github.com/KeyLiaoHPC/TPBench
cd TPBench

# Configure and build
cmake -B build
cmake --build build --config Release

# Run tests to verify the build
cd build && ctest
```

### CMake help target (`tpb_cmake_help`)

After configuring, you can print documented `TPB_*` cache options and the kernel catalog (CPU and ROCm names, tags, and compile prerequisites) without compiling:

```bash
cmake --build build --target tpb_cmake_help
```

### Build Output

After successful build, the directory structure under `build/`:

```
build/
├── bin/
│   ├── tpbcli                  # Main CLI executable
│   └── tpbcli-pli-launcher     # PLI kernel launcher
├── lib/
│   ├── libtpbench.so           # Core TPBench library
│   └── libtpbk_stream.so       # STREAM kernel shared library
└── etc/
    └── yaml/               # Configuration files
```

### Install TPBench

```bash
cmake --install build --prefix /path/to/install
```

Default install paths:
- Binaries: `<prefix>/bin/`
- Libraries: `<prefix>/lib/`
- Headers: `<prefix>/include/`

## Sec. 2 Building with OpenMP Multi-threaded Parallelism

OpenMP enables multi-threaded execution within a single process for shared-memory parallelism.

### Enable OpenMP

`TPB_ENABLE_OPENMP` adds OpenMP compile and link flags to **kernel** targets that are already selected by `TPB_KERNELS` / `TPB_KERNEL_TAGS`. It does not change which kernels are built.

```bash
cmake -B build -DTPB_ENABLE_OPENMP=ON
cmake --build build --config Release
```

Default is OFF. Omit the option or use `-DTPB_ENABLE_OPENMP=OFF` to leave kernel targets without OpenMP.

### Verify OpenMP Support

After building, check OpenMP is linked:

```bash
$ ldd build/lib/libtpbk_stream.so | grep -i openmp
```

### Runtime OpenMP Configuration

When running with OpenMP-enabled kernels, control threading via environment variables:

```bash
export OMP_NUM_THREADS=16
export OMP_PLACES="{0:4}:4:4"  # 4 places with 4 threads each
./build/bin/tpbcli run --kernel stream --kargs stream_array_size=22369621,ntest=100
```

## Sec. 3 Building with MPI Parallelism and Hybrid Parallelism

MPI (Message Passing Interface) enables distributed-memory parallelism across multiple nodes. Hybrid parallelism combines MPI (inter-node) with OpenMP (intra-node).

### Enable MPI

MPI is configured only when an MPI-conditioned kernel (registry `MPI_C_FOUND`) is among the **selected** kernels. MPI is linked to the kernel `.so` targets only; `libtpbench.so` and `tpbcli` do not link MPI. Use `TPB_MPI_PATH` to point at an installation, or leave it empty for CMake to auto-detect.

```bash
# Example: select an MPI kernel by name (when registered) and set MPI root if needed
cmake -B build -DTPB_KERNELS=stream_mpi -DTPB_MPI_PATH=/path/to/mpi/install/dir

cmake --build build --config Release
```

When such a kernel is selected and MPI is found, the following MPI kernels can be built (if present in the registry):

| Kernel | Description |
|--------|-------------|
| `stream_mpi` | STREAM benchmark with MPI |
| `scale_mpi` | Scale operation with MPI |
| `axpy_mpi` | AXPY (BLAS) with MPI |
| `rtriad_mpi` | Repeat Triad with MPI |
| `sum_mpi` | Reduction sum with MPI |

### Hybrid Parallelism (MPI + OpenMP)

To add OpenMP to built kernels while using MPI kernels:

```bash
cmake -B build -DTPB_KERNELS=stream_mpi -DTPB_MPI_PATH=/path/to/mpi -DTPB_ENABLE_OPENMP=ON
cmake --build build --config Release
```

### Running MPI Kernels

```bash
# Run stream_mpi with 4 MPI processes
./build/bin/tpbcli run --kernel stream_mpi \
    --kargs ntest=100,stream_array_size=43690 \
    --wrapper mpirun --wrapper-args '-np 4'

# Hybrid: 2 MPI processes, 8 OpenMP threads each
export OMP_NUM_THREADS=8
./build/bin/tpbcli run --kernel stream_mpi \
    --kargs ntest=100,stream_array_size=43690 \
    --wrapper mpirun --wrapper-args '-np 2 --bind-to socket' \
    --kenvs OMP_NUM_THREADS=8
```

## Sec. 4 Building with More Kernels

By default, only the `stream` kernel is built. Additional kernels can be selected by name or by tag.

### Kernel Selection Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `TPB_KERNELS` | STRING | `"default"` | Comma-separated kernel names, or `"all"` / `"default"` |
| `TPB_KERNEL_TAGS` | STRING | `""` | Comma-separated tags; kernels matching ANY tag are included |

### Available Kernels and Tags

| Kernel | Default Tags | Description |
|--------|--------------|-------------|
| `stream` | `default`, `bandwidth` | STREAM benchmark |
| `triad` | `bandwidth` | Triad operation (a[i]=b[i]+s*c[i]) |
| `scale` | `bandwidth` | Scale operation |
| `axpy` | `bandwidth` | AXPY (BLAS level 1) |
| `rtriad` | `bandwidth` | Repeat Triad |
| `sum` | `bandwidth` | Reduction sum |
| `staxpy` | `bandwidth`, `stanza` | Stanza AXPY (stencil-like) |
| `striad` | `bandwidth`, `stanza` | Stanza Triad (stencil-like) |

MPI kernels additionally have the `mpi` tag.

### Examples

**Build all CPU kernels:**

```bash
cmake -B build -DTPB_KERNELS=all
cmake --build build --config Release
```

**Build specific kernels:**

```bash
cmake -B build -DTPB_KERNELS=stream,triad,scale
cmake --build build --config Release
```

**Build by tag (all bandwidth kernels):**

```bash
cmake -B build -DTPB_KERNEL_TAGS=bandwidth
cmake --build build --config Release
```

**Build stanza kernels plus stream:**

```bash
cmake -B build -DTPB_KERNELS=default -DTPB_KERNEL_TAGS=stanza
cmake --build build --config Release
```

This builds `stream` (default) + `staxpy` + `striad` (stanza tag).

### Per-Kernel Tag Override

Override default tags for a specific kernel using `TPB_KERNEL_<NAME>_TAGS`:

```bash
# Add 'custom' tag to stream, then select by that tag
cmake -B build -DTPB_KERNEL_STREAM_TAGS=default,bandwidth,custom \
               -DTPB_KERNEL_TAGS=custom
```

### Verify Built Kernels

After configuration, CMake prints a summary:

```
-- TPBench kernel selection (TPB_KERNELS=default):
--   Built:   stream
--   Skipped: triad, scale, axpy, rtriad, sum, staxpy, striad
--   (Use -DTPB_KERNELS=all or -DTPB_KERNEL_TAGS=... to enable more)
```

List available kernels at runtime:

```bash
./build/bin/tpbcli kernel list
```

### Kernel Compile History

Different compiler flags produce different `libtpbk_<name>.so` binaries and therefore different KernelIDs. TPBench can record each build as a separate variant in rafdb.

| Option | Default | Description |
|--------|---------|-------------|
| `TPB_RECORD_KERNEL_COMPILE_HISTORY` | ON | Post-build: register kernel and write `variation`/`compilation`/`dependency` metadata |
| `TPB_KERNEL_CFLAGS` | (empty) | Extra C flags for kernel targets (e.g. `-O2`, `-O3`) |

Example: build two `stream` variants into one workspace:

```bash
export TPB_WORKSPACE=$HOME/my-tpbench-ws

cmake -B build-o2 -DTPB_KERNELS=stream -DTPB_KERNEL_CFLAGS=-O2
TPB_WORKSPACE=$HOME/my-tpbench-ws cmake --build build-o2 --target tpbk_stream

cmake -B build-o3 -DTPB_KERNELS=stream -DTPB_KERNEL_CFLAGS=-O3
TPB_WORKSPACE=$HOME/my-tpbench-ws cmake --build build-o3 --target tpbk_stream

./build/bin/tpbcli kernel get -v --kernel stream
```

When a new build replaces `lib/libtpbk_<name>.so`, the previous `.so` is moved to `lib/inactive/libkernel_<name>_<kernel_id>.so_bak` and the old KernelID is marked inactive. The dynloader scans only `lib/libtpbk_*.so` (not `lib/inactive/`).

Rebuilding with identical flags yields the same KernelID; metadata is not updated unless `TPB_K_OVERRIDE=1`.

### Post-Install Kernel Compilation

After TPBench is installed, you can build additional kernels without rebuilding the entire project.

**Recommended: `tpbcli kernel init` / `kernel build`**

Functional CMake files: **`${TPB_HOME}/lib/cmake/TPBench/`** (`find_package(TPBench)`).
Skeleton templates: **`${TPB_HOME}/etc/cmake/kernel/`** (`kernel init`).
CPU kernel catalog: **`src/kernels/kernel_list.cmake.in`** (`NAME|TAGS|PATH`).

```bash
export TPB_HOME=/path/to/tpbench
export TPB_WORKSPACE=$HOME/my-tpbench-ws

# Out-of-tree:
tpbcli kernel init --dir ./mykern --kernel mykern
tpbcli kernel build --dir ./mykern --kernel mykern --cflags "-O2"

# From installed registry tree (default --dir = TPB_HOME):
tpbcli kernel build --kernel 'stream,triad'
tpbcli kernel build --kernel-tag stanza
tpbcli run --kernel mykern --kargs n=10
```

**Manual CMake project (alternative):**

**CMakeLists.txt for a new kernel:**

```cmake
cmake_minimum_required(VERSION 3.16)
project(MyKernel C)

find_package(TPBench REQUIRED)

tpbench_add_kernel(
    NAME mykern
    SOURCES tpbk_mykern.c
)
```

**Build and install the new kernel:**

```bash
cmake -B build -DTPBench_DIR=/path/to/tpbench/lib/cmake/TPBench
cmake --build build
cmake --install build --prefix /path/to/tpbench
```

Or use **`tpbcli kernel build --dir ./mykern --kernel mykern [--tpb-home /path/to/tpbench]`** to install directly into **`${TPB_HOME}/lib`** and register compile metadata.

The new `libtpbk_mykern.so` will be discovered by `tpbcli` at runtime via the PLI scanning mechanism. Execution uses `tpbcli-pli-launcher`, which calls `tpbk_mykern_entry()` inside the `.so`.

## Summary of Build Options

| Option | Values | Default | Description |
|--------|--------|---------|-------------|
| `TPB_ENABLE_OPENMP` | ON/OFF | OFF | Add OpenMP to built kernel targets |
| `TPB_MPI_PATH` | path | empty | MPI root for selected MPI kernel targets only (not linked to libtpbench; empty = auto-detect) |
| `TPB_ROCM_PATH` | path | empty | ROCm root when a `rocm`-tagged GPU kernel is selected (empty = auto-detect) |
| `TPB_USE_AVX512` | ON/OFF | OFF | Enable AVX-512 instructions |
| `TPB_USE_AVX2` | ON/OFF | OFF | Enable AVX2 instructions |
| `TPB_USE_KP_SVE` | ON/OFF | OFF | Enable ARM SVE instructions |
| `TPB_KERNELS` | names/all/default | "default" | Select kernels by name |
| `TPB_KERNEL_TAGS` | comma-separated tags | "" | Select kernels by tag |
| `TPB_SHOW_DEBUG` | ON/OFF | OFF | Enable kernel debug logging |
| `TPB_RECORD_KERNEL_COMPILE_HISTORY` | ON/OFF | ON | Post-build kernel compile metadata registration |
| `BUILD_TESTING` | ON/OFF | ON | Build test suite |
| `CMAKE_INSTALL_PREFIX` | path | /usr/local | Installation directory |
